# AI Kanban Desk

`AI Kanban Desk` — это веб-приложение для управления задачами команды с упором на скорость планирования и прозрачность статусов.  
Проект объединяет классическую канбан-доску, диаграмму Ганта, личные/командные todo и AI-помощника для разбора задач и контекстного чата.

## Что делает проект

- Управляет жизненным циклом задач: от бэклога до выполнения.
- Поддерживает роли пользователей (admin/user), авторизацию и привязку устройства.
- Позволяет работать с задачами в разных представлениях:
  - `Board` (канбан)
  - `Gantt` (сроки и зависимости)
  - `Todo` (персональный список)
- Умеет разбирать текст в структурированные задачи через AI.
- Даёт AI-чат по актуальному состоянию проекта (на основе текущих задач).
- Имеет интеграцию с Telegram-ботом для сценариев автоматизации.

## Верхнеуровневый функционал

- **Канбан-доска**
  - колонки спринта и глобального бэклога;
  - drag-and-drop карточек;
  - приоритет, сложность, исполнитель, сроки, зависимости;
  - история изменений задач.
- **Диаграмма Ганта**
  - визуализация задач с датами;
  - отображение зависимостей;
  - фильтры и экспорт CSV.
- **Todo-режим**
  - личные и связанные с задачами записи;
  - быстрый контроль текущих дел.
- **AI-режим**
  - разбор свободного текста в набор задач для доски;
  - режим общения с ИИ по проектному контексту;
  - история чата в рамках сессии с ручной кнопкой очистки.
- **Команда и доступ**
  - пользователи с ролями `admin` / `user`;
  - вход по логину/паролю;
  - API-защита через auth token.

## Технологии

- `Next.js 14` (App Router)
- `React 18` + `TypeScript`
- `Prisma` + `SQLite`
- `Tailwind CSS`
- `grammY` (Telegram bot)
- `OpenAI SDK` (для Yandex Cloud совместимого API)

## Быстрый запуск

### 1) Требования

- `Node.js` 20+
- `npm` 9+

### 2) Установка зависимостей

```bash
npm install
```

### 3) Настройка окружения

Создай локальный env-файл из шаблона:

```bash
copy .env.example .env.local
```

Для macOS/Linux:

```bash
cp .env.example .env.local
```

Минимально для локального запуска важен `DATABASE_URL` (по умолчанию уже подходит для SQLite).  
Для AI/Telegram-функций заполни соответствующие ключи в `.env.local`.

### 4) Создание базы и Prisma client

```bash
npm run db:migrate
npm run db:generate
```

### 5) Запуск приложения

```bash
npm run dev
```

Открой: [http://localhost:3000](http://localhost:3000)  
(если порт занят, Next автоматически предложит другой, например `3001`).

## Полезные команды

- `npm run dev` — запуск dev-сервера
- `npm run build` — production-сборка
- `npm run start` — запуск production-сборки
- `npm run db:migrate` — применить миграции Prisma
- `npm run db:generate` — пересоздать Prisma Client
- `npm run db:studio` — открыть Prisma Studio
- `npm run bot:dev` — запуск Telegram-бота локально

## Переменные окружения

Смотри `.env.example`:

- `DATABASE_URL` — строка подключения к БД (локально SQLite)
- `YANDEX_CLOUD_API_KEY` — ключ AI-провайдера
- `YANDEX_CLOUD_FOLDER` — id каталога
- `YANDEX_CLOUD_MODEL` — модель
- `TELEGRAM_BOT_TOKEN` — токен Telegram-бота
- `TELEGRAM_WEBHOOK_SECRET` — секрет вебхука

## Структура проекта (кратко)

- `app/` — страницы и API-роуты Next.js
- `components/` — UI и доменные компоненты
- `lib/` — инфраструктурные утилиты (auth, prisma, i18n, constants)
- `hooks/` — клиентские хуки
- `prisma/` — схема и миграции базы
- `src/bot/` — Telegram-бот

## Замечания

- В репозитории используются локализации интерфейса (русский — основной, плюс дополнительные языки).
- Для production рекомендуется:
  - хранить секреты только в переменных окружения;
  - включить CI-проверки и линтер;
  - использовать отдельную БД и резервное копирование.
